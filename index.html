Predicting Tumor Grade from WSI Embeddings

A lightweight, thermal-safe pipeline that explores what pre-extracted whole-slide image embeddings can actually predict on a small clinical cohort. The story: a too-easy task gave fake-perfect results; switching to a harder, balanced target produced an honest, statistically validated finding.

Overview

Foundation models like UNI2 turn whole-slide histopathology images into compact patch-level embeddings. The open question is: how well do those embeddings transfer to clinically meaningful tasks on small TCGA cohorts, where you only have tens of patients per cancer type?

This project benchmarks 13 classical classifiers on TCGA-CHOL (cholangiocarcinoma) WSI embeddings across four prediction targets. The arc of the analysis matters as much as the final number:

  • Step 1 — Naïve setup. Predicting tumor vs normal tissue with slide-level cross-validation gave AUC ≈ 1.00. Too perfect to be useful, and the same 15 participants had matched tumor + normal slides — a textbook leakage risk.
  • Step 2 — Leakage-safe CV. Switched to GroupKFold by participant so the same patient never appeared in both train and test. AUC stayed at 1.00 → the task is simply saturated on UNI2 features, not leaky.
  • Step 3 — Harder target. Tumor grade (G1/G2 low vs G3/G4 high), N = 26, perfectly balanced. KNN-5 achieved AUC = 0.80.
  • Step 4 — Statistical validation. 200-shuffle permutation test produced 0 / 200 null AUCs above the baseline (p = 0.005). The result is real.

Two additional targets were tested and discarded with documentation: BAP1 mutation (AUC 0.64, too class-imbalanced) and PBRM1 mutation (AUC 0.38, anti-correlated noise). The mutation cohort is too small (N = 36, 7–8 positive) for that signal to surface.

Methods

Data

59 whole-slide images from TCGA-CHOL: 39 tumor (sample type 01) and 20 normal (sample type 11) across 44 unique participants, with 15 patients contributing matched tumor + normal pairs. Patch-level embeddings (1,536-dim per patch, variable patches per slide) were pre-extracted with the UNI2 foundation model.

Slide-level aggregation

Each variable-length patch matrix (N × 1536) is aggregated into a fixed slide vector. The default "rich" aggregation concatenates six summary statistics — mean, std, P10, P25, P75, P90 — yielding a 9,216-dim representation. The pipeline also supports plain mean aggregation (1,536-dim).

Models

13 laptop-friendly classifiers, all single-threaded by default:

  • Linear: LogisticRegression (L1 and L2), RidgeClassifier, SGDClassifier (modified Huber), LinearSVC
  • Ensembles: RandomForest, ExtraTrees, GradientBoosting, BaggingClassifier (over LogisticRegression)
  • Other: KNN-3, KNN-5, LinearDiscriminantAnalysis, GaussianNB

Preprocessing pipeline: StandardScaler → PCA (95 % variance retained) → classifier. Preprocessing is refit inside every CV fold to prevent data leakage.

Cross-validation strategy

The pipeline auto-selects the appropriate CV scheme. When matched tumor/normal slides exist for the same participant, GroupKFold (or LeaveOneGroupOut for small cohorts) is used, with the participant ID as the group. This guarantees no participant appears in both train and test. For tumor-only subtasks (grade, mutation) each participant contributes a single slide, so slide-level LeaveOneOut is equivalent to leave-one-patient-out.

Permutation test

To check whether AUC = 0.80 was a small-N fluke, the grade labels were randomly shuffled 200 times. For each shuffle, the same LOO-cross-validated KNN-5 pipeline was run end-to-end and the resulting AUC recorded. The empirical p-value is the fraction of null AUCs ≥ the baseline, with a +1 / N+1 correction (Phipson & Smyth 2010).

Thermal-safe design. The pipeline was built around a laptop that overheats easily. H5 files are loaded one at a time with explicit cooldown pauses; models default to n_jobs = 1; progress is checkpointed every 5 files so a crash never costs more than ~30 seconds of work.

Survival analysis (auxiliary)

As a side analysis, five survival models were fit on the full 36-participant cohort with available survival data, again using GroupKFold by participant. Random Survival Forest reached C-index = 0.714 ± 0.220; Coxnet variants clustered around 0.54. Useful as an honest survival benchmark, though variance is high at this cohort size.

Code & reproducibility

Setup

pip install numpy pandas scipy scikit-learn h5py matplotlib seaborn
pip install scikit-survival lifelines  # optional, for survival models

Run the headline analysis

# Tumor grade — the validated result
python lightweight_wsi_classifier.py \
    --aggregated-csv results_lightweight/aggregated_embeddings.csv \
    --target grade \
    --output-dir results_lightweight \
    --no-survival

Other targets

# Default tumor vs normal (will hit AUC = 1.0)
python lightweight_wsi_classifier.py --embeddings-dir ./embeddings

# Mutation status (binary mut vs wt for any gene with a mut_GENE column)
python lightweight_wsi_classifier.py --target mutation:BAP1 \
    --aggregated-csv results_lightweight/aggregated_embeddings.csv

# Survival quartile (short Q1 vs long Q4)
python lightweight_wsi_classifier.py --target survival_quartile \
    --aggregated-csv results_lightweight/aggregated_embeddings.csv

Force a specific CV strategy

--cv-strategy auto              # leakage-safe by default (recommended)
--cv-strategy group_kfold       # explicit GroupKFold by participant
--cv-strategy logo              # LeaveOneGroupOut by participant
--cv-strategy stratified_kfold  # slide-level (leaky if paired)
--cv-strategy loo               # slide-level LOO (leaky if paired)
