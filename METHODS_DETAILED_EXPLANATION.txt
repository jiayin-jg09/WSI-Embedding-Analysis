1. BACKGROUND: PATCH EMBEDDINGS AND AGGREGATION
================================================================================

A Whole Slide Image (WSI) is a gigapixel-resolution digital scan of a tissue
slide. Because WSIs are too large to process as a single image, they are
divided into thousands of small patches (256 x 256 pixels at 20x magnification).

Each patch is passed through the UNI2 foundation model (a Vision Transformer
pre-trained on over 100 million pathology images), which outputs a 1,536-
dimensional feature vector (embedding) for each patch.

The PROBLEM: Each slide has a different number of patches (typically 1,000 -
30,000), so we cannot directly compare slides. We need a fixed-size
representation.

The SOLUTION: Aggregation methods reduce the variable-length set of patch
embeddings (N x 1,536) into a single fixed-size slide-level vector.

Conceptual Example:
    Slide A: 5,000 patches x 1,536 dims  -->  aggregation  -->  1 x 1,536
    Slide B: 12,000 patches x 1,536 dims -->  aggregation  -->  1 x 1,536
    Now both slides can be compared directly.

Different aggregation methods capture different properties of the patch
distribution:
    - Central tendency methods capture the "typical" patch
    - Dispersion methods capture tissue heterogeneity
    - Extreme value methods may capture rare/unusual regions (e.g., mitoses)
    - Mega-vectors combine multiple perspectives into one representation


================================================================================
2. AGGREGATION METHODS (43 TOTAL)
================================================================================

For all methods below, the input is a matrix of shape (N, 1536) where N is the
number of patches and 1536 is the UNI2 embedding dimension. The output is a
vector. Standard methods output (1536,); mega-vectors output larger dimensions.

--------------------------------------------------------------------------------
2a. CENTRAL TENDENCY (6 methods) - Output: 1,536 dims each
--------------------------------------------------------------------------------
These methods summarize the "typical" patch in a slide.

Method              Formula / Description                           Robustness
-----------------   ----------------------------------------------- ----------
mean                Arithmetic mean: sum(x_i) / N                   Low
                    Most common aggregation. Sensitive to outliers.
                    Treats all patches equally.

median              50th percentile per dimension                    High
                    More robust to extreme patches. Captures the
                    "central" patch without outlier influence.

trimmed_mean        Mean after removing top/bottom 10% of values    Medium
                    Trims extreme patches before averaging.
                    Parameter: proportion=0.1 (10% each tail).

winsorized_mean     Mean after capping extremes at 10th/90th        Medium
                    percentile. Unlike trimmed mean, replaces
                    extremes rather than removing them.
                    Parameter: limits=(0.1, 0.1).

geometric_mean      Product of values raised to 1/N power           Low
                    Emphasizes multiplicative relationships.
                    Values shifted to positive domain first.
                    Better for log-normal distributions.

harmonic_mean       N / sum(1/x_i)                                  Low
                    Emphasizes smaller values. Useful when
                    smaller embeddings are more informative.
                    Values shifted to positive domain first.


--------------------------------------------------------------------------------
2b. DISPERSION / SPREAD (7 methods) - Output: 1,536 dims each
--------------------------------------------------------------------------------
These methods capture how much variation exists across patches -- i.e., how
heterogeneous the tissue is. Higher values = more diverse tissue composition.

Method              Formula / Description                           Interpretation
-----------------   ----------------------------------------------- ----------
std                 Standard deviation: sqrt(var(x))                 General spread
                    Overall patch variability per dimension.

variance            Variance: mean((x_i - mean)^2)                  Spread (squared)
                    Same as std^2. Amplifies large deviations.

range               Max - Min per dimension                          Total spread
                    Sensitive to single extreme patches.

cv                  Coefficient of Variation: std / mean             Relative spread
                    Normalized spread. Allows comparison across
                    dimensions with different magnitudes.

iqr                 Interquartile Range: P75 - P25                   Robust spread
                    Spread of the middle 50% of patches.
                    Unaffected by extreme outlier patches.

mad                 Median Absolute Deviation                        Very robust
                    median(|x_i - median(x)|)
                    Most robust spread measure. Resistant to
                    up to 50% outlier contamination.

mid_range           (Max + Min) / 2 per dimension                    Center of extremes
                    Midpoint between extreme patches.
                    Sensitive to outliers but captures range center.


--------------------------------------------------------------------------------
2c. DISTRIBUTION SHAPE (3 methods) - Output: 1,536 dims each
--------------------------------------------------------------------------------
These methods capture the shape of the patch distribution, not its location
or spread. They reveal asymmetry and tail behavior.

Method              Formula / Description                           Interpretation
-----------------   ----------------------------------------------- ----------
entropy             Shannon entropy of discretized values            Disorder/diversity
                    H = -sum(p_i * log(p_i))
                    Per-dimension: values binned into 50 bins.
                    High entropy = patches spread uniformly.
                    Low entropy = patches clustered together.

skewness            Third moment: E[(x - mean)^3] / std^3           Asymmetry
                    Positive: long right tail (few high patches).
                    Negative: long left tail (few low patches).
                    Zero: symmetric distribution.

kurtosis            Fourth moment: E[(x - mean)^4] / std^4 - 3      Tail heaviness
                    Positive (leptokurtic): heavy tails, sharp peak.
                    Negative (platykurtic): light tails, flat peak.
                    Zero: normal-like distribution.


--------------------------------------------------------------------------------
2d. PERCENTILES (7 methods) - Output: 1,536 dims each
--------------------------------------------------------------------------------
Percentile methods select specific points in the patch distribution. Together,
they capture the full shape without assuming any functional form.

Method              Percentile   Interpretation
-----------------   ----------   ------------------------------------------
percentile_5        P5           Lower tail boundary (near-minimum)
percentile_10       P10          Low-end patches
percentile_25       P25 (Q1)     First quartile
percentile_50       P50 (median) Central tendency
percentile_75       P75 (Q3)     Third quartile
percentile_90       P90          High-end patches
percentile_95       P95          Upper tail boundary (near-maximum)

Why multiple percentiles?
    - P25 and P75 together capture the interquartile range (IQR)
    - P5 and P95 capture the 90% range
    - Comparing P50 to mean reveals skewness
    - Percentiles are robust to outliers (unlike min/max)


--------------------------------------------------------------------------------
2e. EXTREME VALUES / TOP-K (8 methods) - Output: 1,536 dims each
--------------------------------------------------------------------------------
These methods focus on specific subsets of patches rather than the whole slide.
Useful when the most informative patches are rare (e.g., tumor cells, mitotic
figures, immune hotspots).

Method              Description                                     Rationale
-----------------   ----------------------------------------------- ----------
max                 Maximum value per dimension                      Single most extreme
                    Captures the most activated feature per dim.

min                 Minimum value per dimension                      Single least active
                    Captures the least activated feature per dim.

top5_mean           Mean of top 5% patches by L2 norm               High-signal patches
                    L2 norm = sqrt(sum(x_i^2)).
                    Selects patches with strongest overall signal.

top10_mean          Mean of top 10% patches by L2 norm              Broader high-signal
                    Same as top5 but more inclusive.

bottom5_mean        Mean of bottom 5% patches by L2 norm            Background/low-signal
                    Captures patches with weakest signal.
                    May represent background tissue or fat.

bottom10_mean       Mean of bottom 10% patches by L2 norm           Broader low-signal
                    Same as bottom5 but more inclusive.

top5_mean_L1        Mean of top 5% patches by L1 norm               Sparse high-signal
                    L1 norm = sum(|x_i|).
                    L1 emphasizes sparsity vs L2's overall magnitude.

median_patch        Single patch closest to median embedding         Representative patch
                    Finds the actual patch nearest to the
                    element-wise median. Returns a real patch
                    embedding (not a synthetic average).

Note on L2 vs L1 norms for selection:
    L2 norm (Euclidean): sqrt(x_1^2 + x_2^2 + ... + x_1536^2)
        - Sensitive to large individual values
        - Selects patches with strong activation in any dimension
    L1 norm (Manhattan): |x_1| + |x_2| + ... + |x_1536|
        - Treats all dimensions equally
        - Selects patches with broad (non-sparse) activation


--------------------------------------------------------------------------------
2f. WEIGHTED AGGREGATIONS (2 methods) - Output: 1,536 dims each
--------------------------------------------------------------------------------
These methods assign different importance weights to different patches before
averaging. The intuition: not all patches are equally informative.

Method                  Description                                 Weighting
---------------------   ------------------------------------------- ---------
entropy_weighted_mean   Mean weighted by inverse patch entropy       Consistency
                        Per-patch entropy computed from 50 bins.
                        Weight = 1 / (entropy + epsilon).
                        Low-entropy patches (coherent features)
                        get higher weight. Noisy/ambiguous patches
                        get lower weight.

variance_weighted_mean  Mean weighted by inverse patch variance      Stability
                        Per-patch variance computed across 1536 dims.
                        Weight = 1 / (variance + epsilon).
                        Low-variance patches (stable features)
                        get higher weight. Patches with extreme
                        activation patterns get lower weight.


--------------------------------------------------------------------------------
2g. ENERGY / MAGNITUDE (2 methods) - Output: 1,536 dims each
--------------------------------------------------------------------------------
These methods capture the overall magnitude of the embedding signal.

Method              Formula                                         Interpretation
-----------------   ----------------------------------------------- ----------
rms                 Root Mean Square:                                Signal strength
                    sqrt(mean(x_i^2)) per dimension.
                    Like mean but emphasizes larger values.
                    Always >= mean (equality when all equal).

sum_abs             Sum of absolute values per dimension             Total magnitude
                    sum(|x_i|) per dimension.
                    Captures total signal regardless of sign.


--------------------------------------------------------------------------------
2h. MEGA-VECTORS (8 methods) - Output: 4,608 to 10,752 dims
--------------------------------------------------------------------------------
Mega-vectors concatenate multiple aggregation methods into a single higher-
dimensional representation. The idea: different methods capture different
aspects of the tissue, so combining them gives a richer representation.

TRADEOFF: More information vs. higher dimensionality (curse of dimensionality).
PCA is applied before modeling to reduce dimensions.

Method                  Components Concatenated              Output Dims
---------------------   -----------------------------------  -----------
mega_mean_minmax        mean + min + max                     4,608
                        (3 x 1,536)
                        Captures center + extremes.

mega_mean_extremes      mean + top5 + bottom5 +              7,680
                        top10 + bottom10                     (5 x 1,536)
                        Captures center + high/low signals.

mega_mean_percentiles   mean + P10 + P25 + P75 + P90         7,680
                        (5 x 1,536)
                        Captures center + distribution shape.

mega_central_spread     mean + std + var + range + iqr        7,680
                        (5 x 1,536)
                        Captures center + all spread measures.

mega_robust             median + trimmed_mean + iqr +         7,680
                        P25 + P75                            (5 x 1,536)
                        All robust (outlier-resistant) methods.

mega_full               mean + min + max + top5 +             10,752
                        bottom5 + std + range                (7 x 1,536)
                        Most comprehensive standard combo.

mega_distribution       mean + std + skewness + kurtosis      6,144
                        (4 x 1,536)
                        Center + spread + shape parameters.

mega_all_percentiles    P5+P10+P25+P50+P75+P90+P95           10,752
                        (7 x 1,536)
                        Complete percentile profile.

4. SURVIVAL ANALYSIS MODELS (16 TOTAL)
================================================================================

Task: Predict patient survival time (time-to-event) using slide embeddings.
Uses censored data: some patients are still alive at last follow-up.

Evaluation: Concordance Index (C-index) via stratified 5-fold CV.
    C-index = probability that, for a random pair of patients, the one
    predicted to die first actually does die first.
    C-index = 0.5 --> random predictions
    C-index = 1.0 --> perfect predictions

--------------------------------------------------------------------------------

--------------------------------------------------------------------------------
Model                   Key Parameters           How It Works
---------------------   ----------------------   ----------------------------
CoxPH                   --                       Cox Proportional Hazards.
                                                 Assumes log-linear hazard:
                                                 h(t|X) = h0(t) * exp(X*beta).
                                                 The foundational survival model.

CoxnetLasso             l1_ratio=1.0             Cox model with L1 (lasso)
                        alpha_min_ratio=0.01     regularization. Performs
                                                 automatic feature selection.

CoxnetRidge             l1_ratio=0.01            Cox model with near-L2 (ridge)
                        alpha_min_ratio=0.01     regularization. Shrinks all
                                                 coefficients toward zero.

CoxnetElasticNet        l1_ratio=0.5             Cox model with elastic net
                        alpha_min_ratio=0.01     regularization (50% L1 + 50% L2).

5. CLUSTERING & STATISTICAL TESTS
================================================================================

Clustering discovers natural groupings in the data without using labels.

PIPELINE:
1. StandardScaler normalization (zero mean, unit variance)
2. PCA to reduce dimensions (retain 95% variance)
3. K-means clustering (k = 2 to 10, optimal by silhouette score)
4. Hierarchical clustering (Ward linkage)
5. Statistical testing of cluster associations

CLUSTERING METHODS:
    K-means: Partitions data into k clusters by minimizing within-cluster
             sum of squared distances. Tests k=2 through k=10.
             Optimal k chosen by silhouette score.

    Ward:    Hierarchical agglomerative clustering that minimizes total
             within-cluster variance at each merge step. Produces a
             dendrogram showing cluster relationships.

EVALUATION METRICS:
    Silhouette Score (-1 to 1):
        Measures how similar a sample is to its own cluster vs nearest
        cluster. Score > 0.5 indicates reasonable clustering.

    Calinski-Harabasz Index (higher = better):
        Ratio of between-cluster to within-cluster dispersion.
        Higher values indicate better-defined clusters.

STATISTICAL TESTS:
    Chi-square test:
        Tests whether cluster assignments are associated with tumor/normal
        status. Significant p-value means clusters capture biology.

    Log-rank test:
        Tests whether clusters have different survival curves.
        Significant p-value means clustering identifies prognostic groups.

    FDR Correction (Benjamini-Hochberg):
        Corrects p-values for multiple testing across all 43 aggregation
        methods. Controls false discovery rate at alpha = 0.05.

6. CROSS-VALIDATION STRATEGY
================================================================================

Two CV strategies are used to evaluate every (aggregation, model) combination:

1. STRATIFIED 5-FOLD CV (primary):
    - Data split into 5 equal folds, maintaining tumor/normal ratio
    - Each fold used as test set once, train on remaining 4 folds
    - Reports: mean +/- std of metrics across 5 folds
    - Advantage: robust estimate with variance information

2. LEAVE-ONE-OUT CV (supplementary, classification only):
    - N iterations: each sample used as test set once
    - Train on N-1 samples, predict 1 sample
    - Reports: single metric value across all N predictions
    - Advantage: maximum training data per iteration (important for small N)
    - Disadvantage: high variance, computationally expensive

PREPROCESSING WITHIN CV:
    - StandardScaler fitted on TRAINING fold only (prevents data leakage)
    - PCA fitted on TRAINING fold only (prevents data leakage)
    - Test fold transformed using training fold's scaler and PCA
